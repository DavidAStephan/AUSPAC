## What changes in v3.0

- 5 aux .mod files (`dynare/aux/aux_pQ.mod`, `aux_consumption.mod`, `aux_business_inv.mod`, `aux_housing_inv.mod`, `aux_employment.mod`) вҖ” each declares its var_model in terms of STRUCTURAL variables (yhat_au, piQ, pi_m, dln_pcom, etc.) and emits a closed-form expectation formula via `pac.print()`

### Architectural fix: shadow-VAR disconnect eliminated

**Before (Phase R/S)**: 12 shadow var_* variables (`y_gap_var`, `pi_gap_var`, `piQ_hat`, `n_hat`, `c_hat`, etc.) with independent shocks (`eps_var_y`, `eps_var_pi`, ...). Structural shocks like `eps_pQ` did NOT enter the shadow VAR вҖ” they could move the structural `piQ` but the forward PAC expectations were blind to that. Smoking-gun evidence: under Phase S the eps_pQ cost-push IRF was numerically identical across VAR / Hybrid / MCE regimes because the shadow VAR had no channel from `eps_pQ`.

**After (Phase T)**: `pac.print()` writes each PAC equation's expectation as an explicit linear combination of LAGGED STRUCTURAL variables:

```
pac_expectation_pac_pQ = h_0 + h_yhat_au В· yhat_au(-1) + h_i_gap В· i_gap(-1)
                            + h_pi_au_gap В· pi_au_gap(-1) + h_u_gap В· u_gap(-1)
                            + h_yhat_us В· yhat_us(-1) + ...
                            + h_piQ В· piQ(-1) + h_pi_m В· pi_m(-1) + h_dln_pcom В· dln_pcom(-1)
                            + h_piQ_hat В· piQ_hat(-1)
```

`cherrypick` extracts this formula plus the associated h-parameter values into the simulation model. When `eps_pQ` shocks `piQ` in the structural model, the next period's PAC expectation formula in every PAC equation sees `piQ(-1)` directly вҖ” closing the cost-push transmission channel into PAC expectations.

## Replication

```matlab
cd dynare; setup_dynare_path();

% Phase T workflow:
% 1. Estimate aux files + cherrypick (run once)
addpath('aux');
% (run each aux: dynare aux_pQ, then cherrypick to simulation/estimation/pQ/, etc.)

% 2. Build the simulation .mod
addpath(genpath('/Applications/Dynare/6.5-x86_64/matlab/missing'));   % for rows()
aggregate('au_pac_v2.mod', {'stochastic,json=compute'}, pwd, ...
    'simulation/estimation/pQ', 'simulation/estimation/consumption', ...
    'simulation/estimation/business_inv', 'simulation/estimation/housing_inv', ...
    'simulation/estimation/employment', 'simulation/identities');

% 3. Append steady_state_model + shocks + stoch_simul (manual or scripted)

% 4. Run
dynare au_pac_v2;
```
